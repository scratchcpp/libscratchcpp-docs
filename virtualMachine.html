Virtual machine

Scratch projects are compiled to bytecode which is interpreted by the VirtualMachine class.

The opcodes are defined in the Opcode enumeration.

Start instruction

The first instruction of the bytecode is OP_START .

End instruction

The bytecode ends with the OP_HALT opcode which can be also used to stop the script from anywhere in the code.

Constant values

To load a constant value, the OP_CONST opcode can be used.

OP_CONST
0

0 is the index of the constant value. The array of constant values can be set using setConstValues()

To load a null value (zero), use OP_NULL without any arguments.

If statements

If statements can be implemented using OP_IF , OP_ELSE and OP_ENDIF .

(evaluate the condition here)
OP_IF
...
OP_ELSE
...
OP_ENDIF
Note
OP_ELSE is optional.

Loops

All loops end with OP_LOOP_END .

Repeat n times

The repeat loop can be implemented using OP_REPEAT_LOOP .

(load the amount of periods here)
OP_REPEAT_LOOP
...
OP_LOOP_END

Repeat until

Scratch has a while loop, but until is used more frequently, so the VM doesn't support the while loop.

The repeat until loop can be implemented using OP_UNTIL_LOOP and OP_BEGIN_UNTIL_LOOP .

OP_UNTIL_LOOP
(evaluate the condition here)
OP_BEGIN_UNTIL_LOOP
...
OP_LOOP_END
Note
The condition is evaluated after OP_UNTIL_LOOP and the loop starts with OP_BEGIN_UNTIL_LOOP .

Forever

The forever loop can be implemented using OP_FOREVER_LOOP .

OP_FOREVER_LOOP
...
OP_LOOP_END

This is equivalent to:

OP_UNTIL_LOOP
OP_NULL
OP_BEGIN_UNTIL_LOOP
...
OP_LOOP_END

OP_NULL will be evaluated as false.

Variables

Variables are stored in an array similar to constant values.

It can be set using setVariables() .

Set variable at index 0 to loaded value:

(load the value here)
OP_SET_VAR
0

Change variable at index 0 by loaded value:

(load the value here)
OP_CHANGE_VAR
0

Read the variable at index 0:

OP_READ_VAR
0
(use the value in another instruction)

Lists

Lists are stored in an array just like variables. The array can be set using setLists() .

Opcodes for lists can be found in the Opcode enumeration.

Custom functions

Custom functions can be called using OP_EXEC .

Function pointers are stored in an array which can be set using setFunctions() .

Example: print function

OP_CONST
0
OP_EXEC
0
unsigned int print(VirtualMachine *vm) {
std::cout << vm->getInput(0, 1)->toString() << std::endl;
return 1;
}

getInput(0, 1) returns the value at index 0 and 1 is the number of arguments

The function returns the number of arguments.

Note
This is just an example. The OP_PRINT opcode can be used to print a string.
